# Strong Edit Responsibilities

This skill critically evaluates articles. The agent identifies problems; the developer decides what to change.

## Stage Ownership

| Stage | Agent | Developer | Notes |
|-------|-------|-----------|-------|
| 0. Identify core argument | Proposes | **Confirms** | Agent must understand intent before critiquing |
| 1. Evaluate structure | **Leads** | Approves | Agent sees structural patterns |
| 2. Assess relevance | **Leads** | Decides | Agent flags; developer decides what matters |
| 3. Challenge argument | **Leads** | Decides | Agent finds weaknesses; developer judges significance |
| 4. Evaluate readability | **Leads** | Approves | Agent detects friction points |
| 5. Strengthen sections | Proposes | **Leads** | Agent rewrites; developer accepts/modifies/rejects |
| 6. Final assessment | **Leads** | Decides | Agent recommends; developer decides next steps |

**Stage 0 is critical**: The agent must understand what the author is trying to achieve before critiquing. A piece may be "bad" because it's not working—or because the agent misunderstood its purpose.

## Agent Responsibilities

- **Understand before critiquing** - ask what the piece is trying to achieve
- Be specific - "this section is weak" is unhelpful; "paragraph 3 claims X without evidence" is useful
- Distinguish between "unclear to me" and "objectively unclear"
- Propose cuts with quotes, not vague references
- Maintain author's voice when proposing rewrites
- Stop after each stage for approval
- Be honest - don't soften critique to be polite

## Developer Responsibilities

- Clarify the piece's intent and audience
- Judge which critiques matter for this piece
- Decide what to cut, keep, or change
- Provide context agent may lack (audience knowledge, prior work, etc.)
- Final authority on what the piece should say
- Override when agent misunderstands purpose

## Why This Split?

**Agent strengths:**
- Pattern recognition (structural issues, logical gaps)
- Reader perspective (what's confusing, what drags)
- Objective distance from the content
- Can identify issues without ego investment

**Agent limitations:**
- May not understand the author's intent
- Lacks context on audience expertise
- May critique style as substance
- Can over-edit, removing author's voice
- May miss domain-specific conventions

**The critical handoff**: Stage 0 -> Stage 1. If the agent doesn't understand what the piece is trying to achieve, all subsequent critique is potentially misguided.

## The Editor's Role

Strong editing is:
- Honest, not diplomatic
- Specific, not vague
- Subtractive, not additive (prefer cuts to additions)
- Respectful of voice while challenging content

Strong editing is NOT:
- Rewriting the piece into what you would have written
- Imposing a house style
- Softening every critique with "but this is good"
- Adding complexity in the name of rigor

## Difference from Other Skills

| Aspect | review-steps | flesh-out | strong-edit |
|--------|--------------|-----------|-------------|
| Question asked | "Is this correct?" | "What should this say?" | "Does this work?" |
| Primary action | Polish | Generate | Critique |
| Main risk | Missing substance | Meaning distortion | Over-editing |
| Agent posture | Helper | Co-author | Editor |
| Developer posture | Approver | Director | Final judge |
