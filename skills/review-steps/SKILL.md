name: review-steps
description: Steps to do a structured review for a document that is NOT part of a project.
---

When doing a review, if there is no other guidance for a project then follow the following steps.

**Stop after each stage and have changes reviewed with user.**


1. **Review for language and consistency**
   - Fix spelling, grammar, and terminology
   - Ensure consistent usage throughout

2. **Review for conceptual clarity**
   - Complete incomplete sentences
   - Expand acronyms on first use
   - Add missing explanations
   - Identify terms needing glossary entries
   - Identify difficult or imported concepts needing reference entries

3. **Review vs relevant structure**
   - Do the other documents in the same folder have a defined structure.
   - Do the other documents in the same project have a defined structure.

4. **Review vs other solutions and industry best practice** (including web search)
   - Compare against industry frameworks and approaches
   - Identify unique differentiators
   - Ensure proper architecture documentation patterns
   - Note gaps or areas for improvement

5. **Tidy Up**

   - When referring to 3rd party tools and concepts, add markup links directly in the text.
   - Check with the user where to update the glossary - Add terms that needed clarification
